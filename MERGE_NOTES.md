## Merge Conflict Resolution Guide

### Expected Merge Conflicts

When attempting to merge `new_models` into `main`, you may encounter conflicts on large binary model files due to the LFS conversion. Here's how to resolve them:

### Option 1: Accept LFS Conversion (Recommended)
1. **Install and configure Git LFS** (already done in this branch):
   ```bash
   git lfs install
   ```

2. **Convert existing large files to LFS**:
   ```bash
   # Remove existing large files from Git history
   git rm --cached *.pkl *.h5 *.keras *.pdf *.png
   
   # Re-add them to trigger LFS tracking
   git add *.pkl *.h5 *.keras
   git add */*.pdf */*.png
   
   # Commit the LFS conversion
   git commit -m "Convert large model files to Git LFS"
   ```

3. **Merge new_models**:
   ```bash
   git merge origin/new_models
   ```

4. **If conflicts occur on model files**:
   - For each conflicted file, choose the LFS pointer version (theirs)
   - Use: `git checkout --theirs <file>`

### Option 2: Keep Full Files (Not Recommended)
If you prefer to keep large files as regular Git objects:
1. Remove `.gitattributes` 
2. Accept the main branch version of all model files during merge
3. Note: This will result in a large repository size

### Option 3: Clean Merge After Adding Fifth Source Data
1. Add the fifth source data to the directories
2. Run the training script to generate the new model files
3. Merge new_models, accepting their LFS configuration
4. Resolve any conflicts by keeping the newly generated model files
